### 📌 Architecture Principles

- Prefer **clear domain modeling** over temporary patch logic
- Prefer **loose coupling** over direct dependency between business modules
- Prefer **event-driven design** for async and cross-domain workflows
- Prefer **idempotent APIs** for production reliability
- Prefer **observable systems** with metrics, logs and tracing
- Prefer **AI service isolation** for independent evolution of AI capabilities
- Prefer **simple, explicit and maintainable code** over over-engineered abstractions
